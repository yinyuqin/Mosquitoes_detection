# GPU Logistic MIL 基线实验报告

## 1. 实验目的

本实验建立一个计算成本低、结构可解释的蚊子声音检测基线。模型使用预处理后的
MFCC 窗口进行 Logistic 分类，并通过多实例学习（Multiple Instance Learning，MIL）
处理“蚊子声只出现在长片段局部位置”的弱标签问题。

本报告只讨论 HumBugDB 内部分组测试结果。OOD 正负数据没有参与训练、阈值选择或
本次指标计算，因此当前结果不能用于证明跨数据集或真实环境泛化能力。

## 2. 数据预处理

### 2.1 输入与过滤

原始元数据包含 9,295 条标注，每条记录通过唯一 `id` 对应 ZIP 中的 `id.wav`。
预处理逐条读取音频，统一重采样到 8 kHz，并使用以下规则过滤时长：

- 最短时长：0.1 秒；
- 最长时长：60 秒；
- 过滤短片段：104 条；
- 过滤长片段：76 条；
- 读取或特征提取失败：0 条。

最终得到 9,115 个片段：

| 标签 | 数量 | 占比 |
|---|---:|---:|
| 背景（`audio` + `background`） | 2,432 | 26.68% |
| 蚊子（`mosquito`） | 6,683 | 73.32% |
| 合计 | 9,115 | 100% |

### 2.2 MFCC 特征

预处理参数如下：

| 参数 | 数值 |
|---|---:|
| 目标采样率 | 8,000 Hz |
| MFCC 系数 | 13 |
| FFT 窗口 | 512 samples |
| hop length | 256 samples |
| 基础帧窗口 | 64 frames |

每个片段提取 13 维 MFCC、一阶 Delta 和二阶 Delta，共 39 个通道。预处理结果的
帧长度范围为 64～1,920，并被补齐为 64 的倍数。全部 9,115 个特征矩阵均通过
有限值检查，不含 NaN 或无穷值。

预处理还保存两个追踪字段：

- `groups.npy`：原始录音名 `name`，用于防止同源片段跨数据集合；
- `sample_ids.npy`：唯一片段 ID，用于定位预测结果。

## 3. 窗口和补零策略

训练前先去除预处理产生的尾部全零帧，然后按以下方式构建固定长度窗口：

- 窗口长度：64 帧，按 `64 × hop_length / sample_rate` 估算约 2.048 秒；
- 窗口步长：32 帧，按相同口径估算约 1.024 秒；
- 相邻窗口重叠：50%；
- 长片段使用滑动窗口，末尾不足一个步长时补充一个右对齐窗口；
- 短片段训练时将缺失帧随机分配到左右两侧补零；
- 短片段验证和测试时使用居中补零，保证评估可复现。

每个窗口分别计算 39 个通道的均值和标准差，拼接为 78 维输入。标准化统计量
只在当前折的训练窗口上拟合，验证集和测试集不参与均值、方差估计。

## 4. 模型与损失

窗口分类器是一个 PyTorch 线性层：

```text
78 维窗口统计特征 → Linear(78, 1) → window logit → sigmoid → window probability
```

一条片段被视为一个包含多个窗口的 bag。设窗口概率为
\(p_1, p_2, \ldots, p_n\)，片段概率为最高三个窗口概率的平均值：

$$
p_{bag}=\frac{1}{k}\sum_{i\in TopK(p)}p_i,\qquad k=\min(3,n).
$$

片段级概率与片段标签计算二元交叉熵。由于 Top-k 损失只直接更新当前最高概率的
背景窗口，训练还对负 bag 中所有有效窗口计算背景 BCE：

$$
L=L_{bag}+0.5L_{negative\_windows}.
$$

类别权重根据每个训练折的正负样本数计算，避免多数类主导损失。

## 5. 数据切分与调参隔离

切分单位是 CSV 的原始录音名 `name`，不是单个片段：

1. 使用 GroupShuffleSplit 保留约 20% 原始录音组作为独立测试集；
2. 其余数据使用 StratifiedGroupKFold 做五折交叉验证；
3. 共 3,170 个原始录音组，其中训练池 2,536 组、测试集 634 组，组交集为 0；
4. 每一折分别拟合标准化器和模型；
5. 汇总训练集的 OOF 概率，在 0.01～0.99 中选择 F1 最高的阈值；
6. 五个折模型对测试集预测概率取平均，再使用固定阈值评估一次。

最终数据规模：

| 数据集合 | 背景 | 蚊子 | 合计 |
|---|---:|---:|---:|
| 五折训练池 | 1,940 | 5,330 | 7,270 |
| 独立测试集 | 492 | 1,353 | 1,845 |

测试集没有参与模型选择、标准化、早停或阈值选择。

## 6. 训练配置

| 配置 | 数值 |
|---|---:|
| GPU | NVIDIA GeForce RTX 4060 Laptop GPU |
| PyTorch | 2.12.1+cu130 |
| 优化器 | AdamW |
| 学习率 | 0.01 |
| weight decay | 0.0001 |
| batch size | 64 bags |
| 最大 epoch | 50 |
| 早停 patience | 8 |
| Top-k | 3 |
| 负窗口损失权重 | 0.5 |
| 随机种子 | 2026 |
| 最终阈值 | 0.32（训练 OOF F1 最优） |

五折最佳验证结果：

| 折 | 最佳 epoch | 最佳验证损失 |
|---:|---:|---:|
| 1 | 5 | 0.2894 |
| 2 | 3 | 0.2912 |
| 3 | 3 | 0.4318 |
| 4 | 8 | 0.2552 |
| 5 | 11 | 0.2773 |

第 3 折损失明显高于其余折，提示该折可能更难，也可能存在原始录音组之间的
分布差异；类别组成和随机优化同样可能影响单折损失。完整五折
训练在 RTX 4060 Laptop GPU 上约耗时 2 分 30 秒，不包含首次依赖安装和 MFCC
预处理时间。

## 7. 实验结果

阈值相关的 OOF 指标使用同一组 OOF 预测选择阈值并计算，只用于选择工作点，不应视为
完全无偏的泛化估计；ROC-AUC、AP 等无阈值排序指标不受该阈值选择影响。最终结论以
独立测试集为准。

| 指标 | 训练 OOF（同一 OOF 选阈值） | 独立测试集 |
|---|---:|---:|
| Accuracy | 90.59% | 90.79% |
| Balanced Accuracy | 91.04% | 91.13% |
| Precision | 96.87% | 96.83% |
| Recall / Sensitivity | 90.08% | 90.39% |
| Specificity | 92.01% | 91.87% |
| False Positive Rate | 7.99% | 8.13% |
| F1 | 93.35% | 93.50% |
| MCC | 0.7794 | 0.7828 |
| ROC-AUC | 96.22% | 96.35% |
| Average Precision (AP) | 98.56% | 98.47% |
| Brier Score | 0.0853 | 0.0816 |

测试集混淆矩阵：

|  | 预测背景 | 预测蚊子 |
|---|---:|---:|
| 真实背景 | TN = 452 | FP = 40 |
| 真实蚊子 | FN = 130 | TP = 1,223 |

OOF 与测试集主要指标相差很小，没有观察到明显的测试性能坍塌。这里的 AP 使用
`sklearn.metrics.average_precision_score` 计算，不是对 PR 曲线进行梯形积分。
模型在测试集上
检出约 90.4% 的蚊子片段，同时将约 8.1% 的背景片段误报为蚊子。

## 8. 结果解释与限制

### 8.1 可以支持的结论

- MFCC 窗口统计特征对 HumBugDB 蚊子/背景分类具有较强判别力；
- 仅使用一个线性层即可得到 0.9635 ROC-AUC，可作为后续 MLP、CNN 或 ResNet 的
  有竞争力基线；
- OOF 与测试结果接近，当前训练流程没有表现出明显的片段级过拟合；
- MCC 约 0.783、Balanced Accuracy 约 91.1%，说明高 Accuracy 不是只由正样本占
  多数造成的。

### 8.2 不能支持的结论

- 独立测试集仍来自 HumBugDB，不能据此证明 OOD 泛化；
- 测试集蚊子占比约 73.3%，Precision 会高于背景占绝大多数的真实部署环境；
- 8.13% 的片段级背景误报率对长期连续监听仍然偏高；
- 当前阈值以 F1 为目标，并非以实际系统可接受的误报率为目标；
- 线性模型只使用窗口均值和标准差，没有直接利用完整时频纹理和时间结构。

实际应用前应在不参与训练的 OOD 正负数据上测试，并根据训练 OOF 或独立验证集选择
满足目标 FPR 的阈值，例如比较 FPR ≤ 1% 时可达到的 Recall。

## 9. 复现方式

安装依赖：

```powershell
uv venv
uv pip install -r requirements.txt
```

完整预处理：

```powershell
uv run python .\scripts\process_humbugdb_from_zip.py
```

GPU 五折训练：

```powershell
uv run python .\scripts\train_logistic_mil_gpu.py `
  --groups .\data\processed\humbugdb_mfcc\groups.npy
```

默认训练产物写入 `outputs/logistic_mil_gpu/`：

- `fold_1.pt`～`fold_5.pt`：模型、标准化器和窗口配置；
- `metrics.json`：OOF 与测试指标；
- `train_indices.npy`、`test_indices.npy`：数据切分索引；
- `oof_probabilities.npy`、`test_probabilities.npy`：概率输出。

数据和训练产物默认不提交到 Git；代码和本报告记录了本次复现实验所需的主要参数
与结果。固定随机种子不保证跨 CUDA、驱动或 PyTorch 版本逐位一致；需要审计级复现
时还应记录原始文件哈希、完整命令输出和软件环境锁文件。
